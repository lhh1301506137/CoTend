# Community Code of Conduct

CoTend welcomes people with different levels of software experience. Technical disagreement is expected; personal hostility is not.

## Expected conduct

- Be clear, respectful, and specific.
- Critique ideas, evidence, and code rather than people.
- Make space for questions from people who do not read code.
- State uncertainty and correct mistakes without hiding prior context.
- Respect privacy, consent, attribution, and project boundaries.
- Help keep issues and pull requests focused on reproducible work.

## Unacceptable conduct

- Harassment, threats, discrimination, sexualized attention, or personal attacks.
- Publishing private information, credentials, project data, or correspondence without permission.
- Deliberately misleading safety, verification, compatibility, or release claims.
- Repeated disruption, spam, impersonation, or bad-faith demands.
- Retaliation against someone who reports a concern.

## Enforcement

Maintainers may edit or remove contributions, lock discussions, restrict participation, or report abuse to GitHub when behavior violates this policy. Enforcement decisions should be proportionate, documented privately where appropriate, and protect reporters from unnecessary exposure.

For ordinary project concerns, open a focused issue. For platform abuse or a confidential conduct concern, use GitHub's reporting tools rather than posting sensitive details publicly. Security vulnerabilities follow [SECURITY.md](SECURITY.md).

This policy applies in the CoTend repository and in other public spaces when someone is representing the project.
