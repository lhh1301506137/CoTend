# CoTend Support

CoTend is a community-supported pre-release project. Support is provided through GitHub issues on a best-effort basis; there is no paid plan, guaranteed response time, or service-level agreement.

## Where to ask

- Use the bug report form for reproducible installation, discovery, workflow, delivery, or verification failures.
- Use the feature request form for a concrete missing capability or workflow improvement.
- Use [GitHub Discussions](https://github.com/lhh1301506137/CoTend/discussions) only if Discussions is enabled; otherwise open a focused issue.
- Follow [SECURITY.md](SECURITY.md) for vulnerabilities and never disclose them in a public issue.

## Include in a support request

- CoTend version, tag, or commit;
- Codex CLI or Desktop version;
- operating system;
- installation route and the exact CoTend Skill selected;
- minimal reproduction steps;
- expected and actual behavior;
- sanitized output with secrets and private paths removed.

Do not attach a complete private repository, access token, API key, credential file, personal information, or proprietary prompt history.

## Current boundaries

The supported public route is the Codex GitHub Open Beta documented in the README. Public Plugin Directory availability, other AI development tools, and stable backward compatibility are not current support claims.
