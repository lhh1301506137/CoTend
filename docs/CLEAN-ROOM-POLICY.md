# Clean-room 开发与发布边界

```yaml
status: active
authority: product_owner_confirmed
applies_to: repository_and_all_release_artifacts
product_baseline_version: 0.1.0
```

## 仓库边界

- 本仓库是独立公开产品，不是私有上游框架的镜像或发布分支。
- 私有上游只提供抽象理念和经审查的输入，不直接提供可复制文件。
- 原始 intake、本地 AI 治理状态、审查过程和未确认设计候选不属于公开仓库。
- Git 非忽略文件构成当前公开候选集；被 `.gitignore` 排除的本地治理文件不得作为公开规范的来源或实现输入。
- 公开产品结论必须完整写入公开 PRD、能力基线、路线或行为规范，不能依赖本地决策日志才能成立。

## 禁止带入

- 受限第三方框架的隐藏目录、脚本、skills、agents、templates、hooks 或生成文本。
- 私有上游中的个人路径、用户画像原文、谈判材料、内部历史记录和模型私有同步细节。
- 密钥、账号状态、浏览器状态、真实私有数据、未清理截图或本地配置。
- 未经来源和许可证核验的第三方代码、提示词、模板或素材。
- 本地 AI 协议版本、会话轨迹、内部审查日志、计划树和私有来源台账。

## 允许的参考方式

- 可以研究通用概念，例如任务生命周期、项目记忆、状态注入、证据分级和验收脚本。
- 可以对有明确公开来源和许可证的参考实现做固定版本研究，包括目录职责、manifest、核心与适配器边界、调用发现、安装更新卸载、状态生命周期和验证方式；研究产物只能记录可核验事实、权衡与本产品问题。
- 研究公开实现不等于获得复制权或采用权：不得把第三方代码、提示词、模板、生成文件、专有文案或整套目录布局搬入 CoTend，也不得把竞品多数做法直接写成产品事实。
- 公开版的实现、命名、文案、模板和测试必须从本产品需求重新设计并独立编写。
- 外部依赖必须记录来源、版本、许可证、用途和替代方案。

## 来源分级

- `user_owned_original`：能够证明由用户独立创作的机制或材料；仍需去个人化并确认是否混入第三方表达。
- `permissive_external`：许可证允许采用的外部代码或材料；必须保留许可证、归属和修改记录。
- `copyleft_or_restricted`：不适合进入当前 Apache-2.0 发布边界的材料；可以记录公开可核验的架构职责和行为事实，但不得复制代码、表达、结构化模板、生成物或按其布局逐项重建。
- `unknown_provenance`：来源或许可证不能确认的材料；默认禁止进入产品，直到完成审查。
- `private_sensitive`：私人路径、画像、账号、谈判历史和真实项目数据；永不进入公开发布物。

## 操作型 Clean-Room 协议（已确认）

1. 私有上游提炼阶段只产出公开安全的行为规范：用户问题、触发、输出、状态、失败、停止边界和测试。
2. 公开参考实现研究可以额外产出固定来源的架构观察、比较矩阵和待验证问题；它们必须与私有上游提炼分开，并明确不是实现权威。
3. 每份行为规范或参考研究记录来源类别和禁止复制项，不保留上游原句、代码片段、模板正文或可直接复刻的实现材料。
4. 产品实现应在新的实现上下文中进行；实现者只读取获批的公开行为规范、已确认设计和允许的依赖文档，不读取私有上游或受限第三方源文件。参考研究只有经过 CoTend 独立设计确认后才能影响实现。
5. 禁止边看上游文件边逐段改名、翻译或重写，也禁止用自动 diff 驱动“近义复制”。
6. 每个实现切片完成后做来源、许可证和异常相似性审查；不能说明来源的代码、提示词、模板或测试不得发布。
7. 发布前由独立审查者复核高风险来源。AI 审查不能替代必要的法律意见。

## 公开候选检查

首次提交及每次发布前必须从 Git 的非忽略候选集运行自动检查，至少阻止：

- 私有上游名称、本地绝对路径、内部协议版本和本地决策日志引用；
- 未确认的命令、架构、状态布局或安装渠道被写成 active product truth；
- 能力数量、行为索引和规范依赖不一致；
- 公开规范引用被忽略文件作为实现输入。

Canonical check: `python scripts/check_repository.py`.

## 发布门

任何 `git push`、公开发布、包发布或给他人使用前，必须完成：

- 许可证和来源复核。
- 私人路径、密钥、账号、真实数据和内部术语扫描。
- clean-room 差异审查。
- 用户批准发布范围和风险。
- 明确声明 AI UAT 不等于用户最终验收。
