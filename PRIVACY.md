# Privacy Policy

Last updated: July 15, 2026

This policy describes the CoTend open-source repository and skills-only Plugin package. It does not replace the privacy terms of Codex, ChatGPT, GitHub, a model provider, or another tool used with CoTend.

## Data collected by CoTend

CoTend does not provide a hosted account, backend, database, analytics service, telemetry endpoint, advertising system, or CoTend API key. Installing the package alone does not send project files to a CoTend-operated server because no such server is part of the package.

## Data used in your AI development tool

CoTend Skills instruct the host AI development tool to inspect or modify project files, run local tools, and maintain project records when the user authorizes that work. Those files and prompts may be processed by the host tool and its model provider under their settings and policies. CoTend does not control that processing.

Project governance records are stored in the user's project or tool-managed local state according to the active workflow. Users should review generated records and must not place secrets, credentials, or unnecessary personal data in them.

## GitHub interactions

Information submitted to public issues, pull requests, discussions, or commits is public and is processed by GitHub. Security vulnerabilities and sensitive information must not be posted publicly; follow [SECURITY.md](SECURITY.md).

## Third-party material

The package includes two separately licensed companion Skills. Their source and license information is listed in [THIRD-PARTY-SOURCES.json](THIRD-PARTY-SOURCES.json) and [THIRD-PARTY-NOTICES.md](THIRD-PARTY-NOTICES.md). The package does not add a third-party hosted service, but the host AI tool may use its own services.

## Choices and retention

Users control the local project files they keep or delete, subject to their own backup and version-control systems. Removing the CoTend Plugin does not intentionally delete user-owned project records. GitHub content is retained under GitHub's policies and repository history.

## Changes and questions

Material policy changes will be recorded in repository history. For a non-sensitive privacy question, open a GitHub issue and do not include private data. For a security or confidentiality concern, use the private route in [SECURITY.md](SECURITY.md).
